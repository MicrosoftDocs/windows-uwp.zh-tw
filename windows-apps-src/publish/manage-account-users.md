# 管理帳戶使用者


您可以使用 Azure Active Directory，將使用者新增到您的開發人員中心帳戶。 為每位使用者指派角色，為他們提供一組帳戶的特定權限。 您也可以將一個角色指派給一個使用者群組，或者指派給 Azure AD 應用程式。

> 
            **重要** 若要新增和管理帳戶使用者，您必須先將開發人員中心帳戶與組織的 Azure Active Directory 產生關聯。 這要求您使用[全域管理員](http://go.microsoft.com/fwlink/?LinkId=746654)帳戶來登入 Azure AD。 一旦建立此關聯之後，您就無法在不連絡支援的情況下移除它。

 

## 將開發人員中心帳戶與組織的 Azure Active Directory 產生關聯


Windows 開發人員中心會針對多使用者管理與 角色指派來運用 Azure Active Directory。 如果您的組織已經使用 Office 365 或其他 Microsoft 所提供的商務服務，您就已經具備 Azure AD。 否則，您可以從開發人員中心免費建立新的 Azure AD。 

請注意，只有一個開發人員中心帳戶可以與一個 Azure AD 相關聯。 同樣地，只有一個 Azure AD 可以與一個開發人員中心帳戶相關聯。

> 
            **附註** 如果使用者屬於您組織的 Azure AD (或者，如果您為他們建立新的 Azure AD 帳戶)，您只能將他們新增到您的開發人員中心帳戶。 您無法透過使用者個人的 Microsoft 帳戶，將他們新增到開發人員中心帳戶。

將開發人員中心帳戶關聯至組織現有的 Azure AD：

1.  移至您的 \[帳戶設定\]，然後按一下 \[管理使用者\]。
2.  按一下 \[將 Azure AD 關聯至您的開發人員中心帳戶\] 按鈕。
3.  登入您的 Azure AD 帳戶。 這個帳戶必須具備[全域管理員](http://go.microsoft.com/fwlink/?LinkId=746654)權限，才能設定關聯。
4.  檢閱 Azure AD 帳戶的組織和網域名稱。 若要完成關聯，請按一下 \[確認\]。
5.  如果關聯成功，則您已經準備好在帳戶的 \[管理使用者\] 頁面上新增和管理帳戶使用者，如下一節所述。
  
建立全新的 Azure AD 以關聯至您的開發人員中心帳戶︰
1.  移至您的 \[帳戶設定\]，然後按一下 \[管理使用者\]。
2.  按一下 \[建立新的 Azure AD\] 按鈕。
3.  輸入新 Azure AD 的目錄資訊︰
- 
            **網域名稱**︰我們會連同 “.onmicrosoft.com” 針對您的 Azure AD 網域使用的唯一名稱。 例如，如果您輸入 “example”，您的 Azure AD 網域就是 “example.onmicrosoft.com”。 
- 
            **連絡人電子郵件**：我們可以在需要時針對帳戶相關資訊與您連絡的電子郵件地址。
- 
            **全域管理員使用者帳戶資訊**：您想要針對新的系統管理員帳戶使用的名字、姓氏、使用者名稱及密碼。 
4.  按一下 \[建立\] 來確認新網域和帳戶資訊。
5.  使用您新的 Azure AD 全域管理員使用者名稱與密碼來登入，開始在您帳戶的 \[管理使用者\] 頁面上新增和管理其他的帳戶使用者，如下節所述。


> 
            **重要** 將您的開發人員中心帳戶關聯至 Azure AD 之後，一律需要使用 Azure AD 全域管理員帳戶 (非個人的 Microsoft 帳戶) 來登入開發人員中心，以便新增和管理帳戶使用者。

## 新增和管理帳戶使用者、群組及 Azure AD 應用程式


一旦建立關聯之後，您就能將使用者、群組及 Azure AD 應用程式新增到帳戶。 您也可以變更角色、編輯帳戶詳細資料或移除使用者。

> 
            **附註** 如果您的組織使用[目錄整合](http://go.microsoft.com/fwlink/p/?LinkID=724033)來將內部部署目錄服務和您的 Azure AD 進行同步處理，您將無法在開發人員中心建立新的使用者、群組或 Azure AD 應用程式。 您 (或您內部部署目錄中的其他系統管理員) 將需要在內部部署目錄中直接建立它們，然後您才能在開發人員中心看見和新增它們。

管理使用者時，請記住下列資訊：

-   所有的開發人員中心使用者都必須在組織的 Azure AD 中擁有使用中的帳戶。
-   在開發人員中心建立**新的**使用者或群組，也會將它們新增到組織的 Azure AD。
-   在開發人員中心變更使用者或群組的名稱，將會在組織的 Azure AD 中進行這些變更。
-   使用者 (包括群組和 Azure AD 應用程式) 將能使用與其指派角色相關聯的權限來存取完整的開發人員中心帳戶。 您無法限制使用者的存取，因此，他們只能使用特定的 app 和 (或) IAP。
-   您可以藉由選取多個角色，來允許使用者、群組或 Azure AD 應用程式具備一個以上之角色功能的存取權。
-   擁有特定角色的使用者可能也是擁有不同角色之群組的一部分。 在此情況下，使用者將具備與這兩個角色相關聯之功能的存取權。

### 角色和權限

您新增到帳戶的每個使用者、群組或 Azure AD 應用程式至少必須指派下列其中一個角色。 每個角色都具備一組特定權限，以便在帳戶內執行特定功能。

> 
            **附註** 帳戶的擁有者是第一位使用 Microsoft 帳戶建立它的人 (而不是任何透過 Azure AD 新增它的使用者)。 這個帳戶擁有者是唯一具備帳戶完整存取權的人，包括能夠刪除 app、建立和編輯所有帳戶使用者，以及變更所有財務和帳戶設定。 在 Microsoft Visual Studio 中建立應用程式套件時，必須使用用於建立帳戶的 Microsoft 帳戶。

| 角色                 | 說明              |
|----------------------|--------------------------|
| 管理員              | 具備帳戶的完整存取權，但變更稅務和支付設定例外。 這包括管理開發人員中心的使用者，但請注意，建立和刪除使用者的能力取決於帳戶在 Azure AD 中的權限。 也就是，如果為使用者指派管理員角色，但其不具組織 Azure AD 中的系統管理員權限，則他們將無法建立新的使用者或從目錄刪除使用者 (但他們可以變更使用者的開發人員中心角色)。 |
| 開發人員            | 可以上傳套件並提交 app 和 IAP，而且可以檢視[使用方式報告](usage-report.md)來取得遙測詳細資料。 無法檢視財務資訊或帳戶設定。                                                                                                                                                                                                                                                                                                                     |
| 商務參與者 | 可以存取財務資訊，而且可以設定定價詳細資料。 無法建立或提交新的 app 和 IAP，或變更帳戶設定。                                                                                                                                                                                                                                                                                                                                                              |
| 財務參與者  | 可以檢視[支付報告](payout-summary.md)。 無法對 app、IAP 或帳戶設定進行任何變更。                                                                                                                                                                                                                                                                                                                                                                                 |
| 行銷人員             | 可以[回應客戶評論](respond-to-customer-reviews.md)以及檢視非財務性[分析報告](analytics.md)。 無法對 app、IAP 或帳戶設定進行任何變更。                                                                                                                                                                                                                                                                                                            |

> 
            **附註** 具有管理員或開發人員角色的使用者，可以透過儀表板提交 app。 不過，當在 Visual Studio 中建立 app 套件時，必須使用用來開啟開發人員帳戶的 Microsoft 帳戶，而不是 Azure AD 帳戶。

### 新增和管理帳戶使用者

若要識別您想要新增到開發人員中心帳戶的使用者並為他們指派角色，可按一下 \[新增使用者\]。

您可以從組織的目錄新增一或多位使用者到您的開發人員中心帳戶。 請注意，當您同時新增一位以上的使用者時，必須指派相同的角色。 如果您想要新增使用者但為他們指派不同的角色，請針對每個角色重複執行下列步驟。

**從組織的目錄新增使用者**

1.  從 \[管理使用者\] 頁面，按一下 \[新增使用者\]。
2.  從出現的清單中選取一或多位使用者。 您可以使用搜尋方塊來搜尋特定的使用者。
3.  當您完成選取使用者時，請按一下 \[新增選取的項目\]。
4.  在 \[角色\] 區段中，選取一或多個角色來指派給這組使用者。
5.  按一下 \[儲存\]。

如果您想要將開發人員中心存取權授與全新的使用者帳戶，您可以在 \[管理使用者\] 區段中建立新的使用者帳戶。 請注意，這將會在組織的目錄中建立新帳戶，而不只是在您的開發人員中心帳戶中建立。

**建立新的使用者帳戶**

1.  從 \[管理使用者\] 頁面，按一下 \[新增使用者\]。
2.  在下一頁，按一下 \[新使用者\]。
3.  輸入新使用者的名字、姓氏及使用者名稱。
4.  在 \[角色\] 區段中，選取一或多個角色來指派給新的使用者。
5.  在 \[群組成員資格\] 區段中，選取任何您想讓新使用者隸屬的群組。
6.  按一下 \[儲存\]。
7.  在確認頁面上，您將會看見新使用者的登入資訊 (包括暫時密碼)。 請確定會記下此資訊，並將它提供給新的使用者，因為您在離開此頁面之後將無法存取該暫時密碼。

您可以在 \[管理使用者\] 區段中，為要新增到開發人員中心帳戶的使用者帳戶進行變更。 請注意，對於使用者名稱或群組成員資格的變更將會反映於組織的目錄中，而不只是在您的開發人員中心帳戶中。 對於使用者角色所做的變更將只會影響他們的開發人員中心存取權。

**編輯使用者帳戶**

1.  從 \[管理使用者\] 頁面，按一下您要編輯的使用者帳戶名稱。
2.  進行下列任何變更：
    -   編輯使用者的名字、姓氏或使用者名稱。 請記住，這些變更將會在組織的目錄中進行。
    -   在 \[角色\] 區段中，選取或取消選取您要針對此使用者新增或移除的角色。
    -   在 \[群組成員資格\] 區段中，選取或取消選取您要讓使用者加入或從中移除的群組。 請記住，這些變更將會在組織的目錄中進行。

3.  按一下 \[儲存\]。

如果您需要變更您已新增到開發人員中心帳戶之使用者帳戶的密碼，可在 \[管理使用者\] 區段中執行此動作。 請注意，這將會變更使用者的目錄密碼，而不只是其開發人員中心存取權的密碼。

**變更使用者的目錄密碼**

1.  從 \[管理使用者\] 頁面，按一下您要編輯的使用者帳戶名稱。
2.  按一下頁面底部的 \[重設密碼\] 按鈕。
3.  隨即會出現確認頁面，顯示該使用者的登入資訊 (包括暫時密碼)。
  > 
            **重要** 請務必列印或複製此資訊，並將它提供給該使用者，因為您在離開此頁面之後將無法存取該暫時密碼。

### 新增與管理群組

當您從組織的目錄將群組新增到開發人員中心帳戶時，隸屬於該群組成員的每位使用者都能使用與群組指派角色相關聯的權限來存取它。 請記住，對於群組 (包括其名稱或成員資格) 所做的任何變更都將反映於組織的目錄中。

請注意，當您同時新增一個以上的群組時，必須指派相同的角色。 如果您想要新增群組但為它們指派不同的角色，請針對每個角色重複執行下列步驟。

**從組織的目錄新增群組**

1.  從 \[管理使用者\] 頁面，按一下 \[新增群組\]。
2.  從出現的清單中選取一或多個群組。 您可以使用搜尋方塊來搜尋特定的群組。
3.  當您完成選取群組時，請按一下 \[新增選取的項目\]。
4.  在 \[角色\] 區段中，選取一或多個角色來指派給這組群組。
5.  按一下 \[儲存\]。

如果您想要將開發人員中心存取權授與全新的群組，您可以在 \[管理使用者\] 區段中建立新的群組。 請注意，這將會在組織的目錄中建立新群組，而不只是在您的開發人員中心帳戶中建立。

**建立新的群組帳戶**

1.  從 \[管理使用者\] 頁面，按一下 \[新增群組\]。
2.  在下一頁，按一下 \[新群組\]。
3.  輸入新群組的顯示名稱。
4.  選取一或多個角色來指派給新的群組。 群組的所有成員都能使用與該角色相關聯的權限來存取您的開發人員中心帳戶
5.  從出現的清單中選取一或多位使用者。 您可以使用搜尋方塊來搜尋特定的使用者。
6.  當您完成選取使用者時，請按一下 \[新增選取的項目\]。
7.  按一下 \[儲存\]。

您可以在 \[管理使用者\] 區段中，為要新增到開發人員中心帳戶的群組帳戶進行變更。 請注意，對於群組名稱或成員資格的變更將會反映於組織的目錄中，而不只是在您的開發人員中心帳戶中。 對於群組角色所做的變更將只會影響它們的開發人員中心存取權。

**編輯群組帳戶**

1.  從 \[管理使用者\] 頁面，按一下您要編輯的群組帳戶名稱。
2.  若要變更群組資訊，請對群組名稱進行任何所需的變更。 請記住，這些變更將會在組織的目錄中進行。
3.  若要變更群組角色，可選取或取消選取您想要套用到群組的角色。
4.  按一下 \[儲存\]。

### 新增和管理 Azure AD 應用程式

您可以允許屬於組織 Azure AD 一部分的應用程式或服務存取您的開發人員中心帳戶。

請注意，當您同時新增一個以上的 Azure AD 應用程式時，必須指派相同的角色。 如果您想要新增群組但為它們指派不同的角色，請針對每個角色重複執行下列步驟。

**從組織的目錄新增 Azure AD 應用程式**

1.  從 \[管理使用者\] 頁面，按一下 \[新增 Azure AD 應用程式\]。
2.  從出現的清單中選取一或多個 Azure AD 應用程式。 您可以使用搜尋方塊來搜尋特定的 Azure AD 應用程式。
3.  當您完成選取 Azure AD 應用程式時，請按一下 \[新增選取的項目\]。
4.  在 \[角色\] 區段中，選取一或多個角色來指派給這組 Azure AD 應用程式。
5.  按一下 \[儲存\]。

如果您想要將開發人員中心存取權授與全新的 Azure AD 應用程式帳戶，可以在 \[管理使用者\] 區段中建立新的使用者帳戶。 請注意，這將會在組織的目錄中建立新帳戶，而不只是在您的開發人員中心帳戶中建立。

**建立新的 Azure AD 應用程式**

1.  從 \[管理使用者\] 頁面，按一下 \[新增 Azure AD 應用程式\]。
2.  在下一頁中，按一下 \[新增 Azure AD 應用程式\]。
3.  輸入新的 Azure AD 應用程式的 \[回覆 URL\]。 使用者可以從此 URL 登入並使用您的 Azure AD 應用程式 (也稱為 App URL 或登入 URL)。 \[回覆 URL\] 的長度不能超過 256 個字元。
4.  輸入新 Azure AD 應用程式的 \[應用程式識別碼 URI\]。 這是 Azure AD 應用程式的邏輯識別碼，是在傳送單一登入要求至 Azure AD 時提供。 請注意，在目錄中的每個 Azure AD 應用程式的 \[應用程式識別碼 URI\] 必須是唯一的，且長度不能超過 256 個字元。
5.  在 \[角色\] 區段中，選取一或多個角色來指派給新的 Azure AD 應用程式。
6.  按一下 \[儲存\]。

您可以在 \[管理使用者\] 區段中，為要新增到開發人員中心帳戶的 Azure AD 應用程式進行變更。 請注意，對於 \[回覆 URL\] 或 \[應用程式識別碼 URI\] 的變更將會反映於組織的目錄中，而不只是在您的開發人員中心帳戶中。 變更角色只會影響開發人員中心內的 Azure AD 應用程式。

**編輯 Azure AD 應用程式**

1.  從 \[管理使用者\] 頁面，按一下您要編輯的 Azure AD 應用程式帳戶名稱。
2.  若要變更 \[回覆 URL\] 或 \[應用程式識別碼 URI\]，請在這裡輸入新的值。 請記住，這些變更將會在組織的目錄中進行。
3.  若要變更 Azure AD 應用程式的角色，可選取或取消選取您想要套用的角色。
4.  按一下 \[儲存\]。

如果您的 Azure AD 應用程式會在 Microsoft Azure AD 讀取或寫入資料，則它需要金鑰。 您可以在開發人員中心編輯 Azure AD 應用程式的資訊來新增其金鑰。 您也可以移除不再需要的金鑰。

**管理 Azure AD 應用程式的金鑰**

1.  從 \[管理使用者\] 頁面，按一下 Azure AD 應用程式的名稱。

    > 
            **提示** 按一下 Azure AD 應用程式的名稱，您就會看到該 Azure AD 應用程式所有使用中的金鑰，包括金鑰的建立和到期日期的資料。 若要移除不再需要的金鑰，請按一下 \[移除\]。

2.  若要新增金鑰，按一下 \[新增金鑰\]。

3.  您將會看到顯示 \[用戶端識別碼\] 和 \[金鑰\] 值的畫面。

    > 
            **重要** 請務必複製此資訊，您離開這個頁面之後就無法再存取此資訊。

4.  如果您想要建立更多金鑰，請按一下 \[新增其他金鑰\]。

### 移除使用者、群組及 Azure AD 應用程式

若要從您的開發人員中心帳戶移除使用者、群組或 Azure AD 應用程式，請在 \[管理使用者\] 頁面上，按一下依其名稱顯示的 \[移除\] 連結。 在確認您想要移除它之後，該使用者、群組，或 Azure AD 應用程式將無法再存取您的開發人員中心帳戶 (除非您稍後再次新增它)。

> 
            **附註** 移除使用者、群組或 Azure AD 應用程式表示它將不再具備您開發人員中心帳戶的存取權。 它不會從組織的目錄中刪除使用者、群組或 Azure AD 應用程式。
