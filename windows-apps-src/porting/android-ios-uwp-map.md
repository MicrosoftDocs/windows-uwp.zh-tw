#適用於 Android 與 iOS 開發人員的 Windows 應用程式概念對應

如果您是具備 Android 或 iOS 技巧和 (或) 程式碼的開發人員，而且您想要移到 Windows 10 和通用 Windows 平台 (UWP)，則此資源擁有您在三個平台之間對應平台功能 (和您的知識) 所需的資訊。

另請參閱[從 iOS 移到 UWP](ios-to-uwp-root.md) 中的移植內容。

|  **編輯控制項的視覺結構。** 自訂控制項的視覺結構，不只是修改屬性 (property) 或屬性 (attribute)，例如將核取方塊文字移到核取方塊的下方。 | Android 中沒有編輯控制項之視覺結構的簡單方法。 | iOS 中沒有編輯控制項之視覺結構的簡單方法。 | 若要自訂控制項的視覺結構，您可以在 XAML 標記中複製並編輯其**[控制項範本](https://msdn.microsoft.com/library/windows/apps/xaml/windows.ui.xaml.controls.controltemplate.aspx)**。<br/><br/>[快速入門：控制項範本](https://msdn.microsoft.com/library/windows/apps/xaml/hh465374.aspx)  |

## 資料

|  **一般概念** | **Android** | **iOS** | **Windows 10** |
|  ------ | ------ | ------ | ------ |
|  **本機 app 資料。** 將與 app 相關的設定和檔案儲存在本機。 | 使用 **openFileOutput** 和 **openFileInput** 可以儲存本機檔案。 使用 **getSharedPreferences** 可存取**共用喜好設定檔案**中的設定。 | 本機檔案可以儲存在**應用程式支援**目錄中，並透過 **NSFileManager** 類別存取。 **喜好設定**檔案中的設定可以透過 **NSUserDefaults** 類別存取。 | **
