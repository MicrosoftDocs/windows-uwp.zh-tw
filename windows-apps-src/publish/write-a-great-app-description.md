# <a name="write-a-great-app-description"></a>撰寫一份出色的應用程式介紹


一份出色的 App 描述可以讓您的 App 在 Microsoft Store 中更為突出，並協助鼓勵客戶下載。 [您提交 app 時輸入的描述](create-app-store-listings.md#description)會顯示在您的 app 市集清單。 前幾行文字可能也會顯示在搜尋結果和市集中列出的演算法清單中。

這裡是一些讓您的 app 描述最為出色的秘訣。

-   **擷取在前幾個句子的注意。** 描述的開頭非常重要，要確定能夠抓住目光。 以價值論點起頭：為什麼潛在客戶應該花時間與金錢取得您的 App？ 和其他 App 相比，選擇您 App 的好處是什麼？ 利用一或兩個句子，以淺顯易懂的語言說明您 App 的獨特吸引力，以及人們會想要取得它的原因。
-   **讓您輕鬆了解您的應用程式。** 吸引客戶注意之後，隨後說明額外的好處、在 app 內購買的商機以及客戶想要知道關於您 app 的其他詳細資訊。 請確定您已遵照 app 發行所在市場之法律規定，提供任何必要的揭露訊息或資訊。
-   **使用清單和短的段落。** 潛在客戶很可能只會快速瀏覽您的 app 描述。 使用簡短的段落和清單將內容分段，可以讓使用者更容易閱讀。

    > [!NOTE]
    > 新增一份[產品功能](create-app-store-listings.md#product-features)也可以快速顯示您的應用程式用途的說明。 這個清單要顯示在 app 描述的正下方。

-   **避免 dry 的語言。** 使用生動的語調撰寫描述。 請確定內容清楚描述 app 的功能，但不要以太枯燥的文字描寫。 對很多 app 而言，輕鬆活潑的語調效果最好。
-   **使用正確的長度。** 好的描述不必太長，但能包含足夠的資訊讓讀者感興趣，並說明 app 的功能。 複雜的 app 需要多點內容說明；簡單的 app 則只需簡單幾個字即可。 大多數情況下，適中的長度大約為 200 多個字，但不會超過 3000 個字。
-   **是清楚了解免費試用版和附加元件。** 如果您提供產品的免費試用，請務必解釋試用版的使用方式，讓客戶能了解哪些功能受到限制。 說明有哪些類型的附加內容也是一個不錯的方法，尤其是當這些選項對您的 App 功能影響很大時。
-   **使用標準的大小寫和標點符號。** 使用全部大寫或不常用的標點符號的描述可能難以閱讀。
-   **別忘了要檢查拼字和文法檢查。** 描述中有很多錯字或不順暢的句子會讓人對應用程式品質有不好的觀感。 記得檢查您的描述 (或讓其他人檢查) 是否有錯誤。
-   **不包含連結或其他位置所屬的資訊。** 在描述欄位中輸入的 URL 不是可按一下的連結，所以不要嘗試加入隱私權原則或支援網站之類的資訊連結。 相反地，加入下列的指定區域中**屬性**您提交的頁面。
-   **請勿使用 HTML 標記。** HTML 或其他程式碼不會被轉譯。 您的描述必須是純文字。
-   **獲得構想，藉由檢視類似的應用程式存放區中的描述。** 參考其他開發人員如何描述他們的 app。 這也有助於您找出 app 的獨特之處，並加以強調。
