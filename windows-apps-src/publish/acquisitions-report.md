# <a name="acquisitions-report"></a>下載數報告


**併購**中的報表[合作夥伴中心](https://partner.microsoft.com/dashboard)可讓您查看誰會取得並安裝您的應用程式，以及 人口統計平台的詳細資訊，以及有關如何顯示資訊 （包括 Windows 10 上的客戶Xbox) 已抵達您的應用程式清單。 您也可以檢視最後一小時或 seventy 兩小時期間的接近即時擷取的資料。 

您可以檢視這項資料在合作夥伴中心，或[將報表下載為](download-analytic-reports.md)要離線檢視。 或者，您可透過程式設計方式使用我們的[分析 REST API](../monetize/access-analytics-data-using-windows-store-services.md) 擷取此資料。

在此報告中，一個**下載數**表示一位新客戶已取得您 app 的授權 (不論是付費或免費取得)。 **安裝**指的是安裝在 Windows 10 裝置上的 app。

> [!IMPORTANT]
> **併購**報表不包含有關退款、 低價、 退款等資料。若要預估您的應用程式會繼續進行，請瀏覽[支出摘要](payout-summary.md)。 在 [**保留**] 區段中，按一下 [**下載保留的交易**] 連結。
>
> 除了頁面檢視資料（如下所述），這份報告不包含有關未登入 Microsoft 帳戶而取得應用程式之客戶的資料。


## <a name="apply-filters"></a>套用篩選

在頁面頂端附近，您可以選取您想要顯示資料的時間週期。 預設選項是 **30D**（30 天），但您可以選擇在 3、6 或 12 個月期間顯示資料，或指定自訂的日期範圍。 近乎即時的資料將會顯示所有選項 (除了**應用程式的累計**資料)。 **1 小時**並**72 小時**時間週期只會套用到**應用程式每日**索引標籤**併購**圖表和**收購**索引標籤**市場**圖表。 

您也可以展開 **\[篩選條件\]**，依市場和/或裝置類型篩選此頁面上的所有資料。

-   **市場**:預設篩選是**所有市場**，但您可以將資料限制為一或多個市場的併購。
-   **裝置類型**:預設值是**所有裝置**。 如果您只想要顯示來自特定裝置類型 (例如個人電腦、主機或平板電腦) 的下載數資料，您可以在此處選擇特定的類型。

下列所有圖表中的資訊將反映您選取的日期範圍和任何篩選條件。 某些區段也可讓您套用其他篩選。


## <a name="acquisitions"></a>下載數

**下載數**圖表會顯示在選取時段內，每日或每週取得數目 (取得您 app 授權的新客戶)。 (當您利用 **\[套用篩選\]** 顯示較長期間的資料時，下載數資料將會以週為單位分組)。取得所做的有效的 Microsoft 帳戶登入的客戶會包含在此圖表。 

根據預設，我們示範**應用程式每日**檢視，其中包括接近即時的資料。 您也可以透過選取 **\[App cumulative\]** (App 累積)，看到您 app 生命週期內的下載數。 這會顯示所有下載數的累計總數 (從您 app 第一次發行開始)。

**總銷售額**針對您的應用程式 （從 10 月 2016-存在) 中還會提供這張圖表，顯示所獲得的 （以美元為單位） 的應用程式銷售的總金額。 請注意，此數量不會考慮任何退款、 低價、 退款等。

您可以選擇性地依據下載是來自用戶端或 web 架構市集，和（或）依據 OS 版本來篩選結果。

> [!NOTE]
> 您也可透過程式設計方式使用我們的[分析 REST API](../monetize/access-analytics-data-using-windows-store-services.md) 中的[取得應用程式下載數](../monetize/get-app-acquisitions.md)方法來擷取此資料。

在 **應用程式每日**檢視，當**30 D**選取時間期間，您可能會看到圓形標記。 這些代表大幅增加或減少中指定的值，我們認為您會想要了解。 顯示圓形的日期表示我們偵測大幅增加或減少相較於當週之前一週結尾。 若要查看更多詳細的變更，將滑鼠停留在圓形上。  

> [!TIP]
> 您可以檢視過去 30 天中的重大變更相關的更多深入資訊[Insights 報告](insights-report.md)。

## <a name="installs"></a>安裝數

**安裝數**圖表顯示在所選時段內，我們偵測到客戶在 Windows 10 裝置 (包括 Xbox One 主機) 上成功安裝您的應用程式的次數。 會顯示總數，以及圖表依天或星期顯示安裝數（根據您已選取的持續時間）。 您可以選擇性地依據特定套件版本篩選結果。

安裝總數包括︰
-   **多個 Windows 10 裝置上進行安裝。** 例如，如果同一位客戶在兩部 Windows 10 電腦和一台 Xbox One 主機上安裝您的應用程式，就算為三個安裝。
-   **重新安裝。** 例如，如果客戶今天安裝您的應用程式，明天解除安裝應用程式，然後在下個月重新安裝應用程式，便算成兩個安裝。

安裝總數不包括或反映︰
-   **非 Windows 10 裝置上進行安裝。** 如果您的應用程式支援較舊的作業系統版本，例如 Windows 8.x 或 Windows Phone 8.x，我們不計算在那些裝置上的任何安裝數。
-   **解除安裝。** 當客戶從裝置解除安裝您的應用程式，我們不會從總安裝數減去該次安裝。
-   **更新。** 例如，如果客戶今天安裝您的應用程式，然後在一星期後安裝應用程式更新，則只計入一次安裝。
-   **預先安裝。** 如果客戶購買已預先安裝應用程式的裝置，我們不會將其計算為一次安裝。
-   **系統起始的安裝。** 如果 Windows 因某些原因自動安裝您的應用程式，我們不會其算為一次安裝。

> [!NOTE]
> 您也可透過程式設計方式使用我們的[分析 REST API](../monetize/access-analytics-data-using-windows-store-services.md) 中的[取得應用程式安裝數](../monetize/get-app-installs.md)方法來擷取此資料。

## <a name="acquisition-funnel"></a>取得漏斗圖

**取得漏斗圖**顯示多少客戶完成漏斗圖的每個步驟，從檢視市集頁面到使用應用程式，以及轉換率。 這項資料可以協助您識別哪些領域可能要提高投資，以增加下載數、安裝數或使用量。

> [!IMPORTANT]
> **取得漏斗圖**只針對過去 90 天的 Windows 10 (包括 Xbox) 客戶顯示資料。

漏斗圖的步驟是︰

- **頁面檢視**:此數字代表您的應用程式存放區清單，包括不是 Microsoft 帳戶登入的檢視總數。 這不會包括選擇不提供這項資訊給 Microsoft 的客戶資料。
- **併購**:新的檢視其存放區清單的 48 小時內取得的授權，您的應用程式 （當其 Microsoft 帳戶登入） 的客戶數目。
- **安裝**:安裝之後取得它的應用程式的客戶數目。
- **使用量**:在安裝它之後會使用應用程式的客戶數目。

您可以選擇性地依據性別和/或年齡層，以及依據自訂行銷活動識別碼來篩選結果。

> [!NOTE]
> 您也可透過程式設計方式使用我們的[分析 REST API](../monetize/access-analytics-data-using-windows-store-services.md) 中的[取得應用程式取得漏斗圖資料](../monetize/get-acquisition-funnel-data.md)方法來擷取此資料。

## <a name="markets"></a>市場

**市場**圖表會依您 App 上市的每個市場顯示所選時段內的下載或安裝總數。 您可以選擇是否要顯示**下載數**或**安裝數**的資料。

您可以使用視覺**地圖**形式檢視這些資料，或切換設定，以**表格**形式檢視資料。 表格形式一次顯示五個市場，依字母排序或依最高/最低下載數或安裝數排序。 您也可以下載資料，一起檢視所有市場資訊。


## <a name="customer-demographic"></a>客戶人數統計

[客戶人數統計] 圖表顯示取得您 app 對象的人數統計資訊。 您可以依特定年齡層和依性別來查看取得對象在選取時段內的下載數。

> [!NOTE]

## <a name="app-page-views-and-conversions-by-channel"></a>依管道的 app 頁面檢視和轉換

**應用程式頁面檢視和通道的轉換**圖表可讓您查看 Windows 10 上的客戶如何抵達的時間所選時段內的應用程式的清單。

在此圖表，*通路*是指客戶到達您 app 清單頁面 (例如，在市集中瀏覽及搜尋、來自外部網站的連結、來自您自訂行銷活動等等的其中一個連結) 的方法。 包含下列的管道類型：

-   **儲存流量：** 客戶瀏覽或搜尋存放區中，當他們檢視您的應用程式清單。
-   **自訂活動：** 客戶接著可使用的連結[自訂活動識別碼](create-a-custom-app-promotion-campaign.md)。
-   **其他：** 客戶遵循網站從外部連結 （不含任何自訂的活動識別碼），以您的應用程式清單，或客戶從搜尋引擎遵循連結以您的應用程式清單。

A*頁面上檢視*表示客戶，檢視您的應用程式存放區清單頁面上，透過 web 為基礎的存放區，或是從 Windows 10 上的市集應用程式。 這包括未登入 Microsoft 帳戶的人員檢視。 部分客戶選擇不分享此資訊給 Microsoft。

*轉換*表示客戶 (使用 Microsoft 帳戶登入) 已經新獲得您 App 的授權 (不論是付費或免費取得)。

頁面檢視數與轉換數並非不重複客戶的計數。 如需轉換率資訊，請參閱[取得漏斗圖](#acquisition-funnel)。

> [!NOTE]
> 客戶可能透過按一下並非由您建立的自訂行銷活動，而連到您的應用程式清單。 我們將工作階段中的每個頁面檢視都加上客戶首度進入市集的行銷活動識別碼。 接著，對於 24 小時內的所有下載，我們會將轉換歸至該行銷活動識別碼。 因此，您可能會看到比您行銷活動識別碼轉換總計更多的轉換總計，以及可能會有頁面檢視為零的轉換或附加元件轉換。

> [!NOTE]
> 您也可透過程式設計方式使用我們的[分析 REST API](../monetize/access-analytics-data-using-windows-store-services.md) 中的[取得應用程式轉換 (依通路)](../monetize/get-app-conversions-by-channel.md) 方法來擷取此資料。
